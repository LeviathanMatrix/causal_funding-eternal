# Demo Scope

## Demo Objective

The Eternal demo is designed to prove that `causal_funding` is more than a token alarm. It is intended to show a credible operating loop that can help a reviewer move from raw on-chain uncertainty to structured decision support within a short time window.

## What the Demo Will Show

The judge-facing demo focuses on the following visible outcomes:

- token or mint input
- fast first-pass analysis
- upstream funding-path attribution
- multi-dimensional risk output
- evidence-backed summary
- agent-generated judgement for operator review

The purpose is to show how a user can move from signal to interpretation to action without needing to manually assemble dozens of disconnected on-chain clues.

## Expected Runtime

The current target is a first-pass structured result in roughly 1-2 minutes per token. This is an actively optimized area, and improving latency remains part of the near-term execution plan.

## Demo Narrative

The demo should communicate four things clearly:

1. Solana requires real-time risk intelligence, not delayed post-mortems.
2. `causal_funding` is not a basic rug checker or static scorecard.
3. The system produces reviewable evidence, not only raw outputs.
4. The architecture supports a realistic path toward institutional deployment.

## What Is Intentionally Not Public

This repo and demo do not expose:

- the full production architecture
- sensitive internal thresholds or weighting logic
- internal data pipelines
- unrestricted public API access

The current external posture is deliberate. Evaluation access can be provided to judges and selected pilot partners, while broader release is deferred until the system is hardened further.

## User Journey

The intended operator flow is:

1. identify a token that requires review
2. run a causal attribution pass
3. inspect multi-dimensional risk outputs
4. review evidence-backed agent conclusions
5. decide whether to proceed, escalate, block, or continue monitoring

## Demo Success Criteria

The Eternal build succeeds if a reviewer can quickly understand:

- why the product matters on Solana
- why the output is more useful than a simple risk score
- why the system can plausibly serve real listing, investment, and trading workflows
- why the product has a believable path to restricted institutional integration
