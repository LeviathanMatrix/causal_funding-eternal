# Eternal Submission Checklist

Use this checklist for each weekly update and final submission.

## Weekly Checklist

- [ ] GitHub demo repo updated with this week's shipped changes
- [ ] 1-minute video recorded (Loom or YouTube)
- [ ] Weekly update posted in Eternal dashboard before cutoff
- [ ] README and docs remain aligned with actual product behavior
- [ ] No sensitive internals or secrets exposed publicly

## Video Checklist

- [ ] Show mint input and run start
- [ ] Show final risk output and decision action
- [ ] Show one evidence highlight
- [ ] State what improved this week
- [ ] State next week's plan

## Final Submission Checklist

- [ ] Public demo repo is clean and understandable
- [ ] Judge access instructions are clear
- [ ] At least one sample redacted report is included
- [ ] Product positioning, scope, and roadmap are consistent
- [ ] Contact channels for judges/partners are active
